# Security Policy

## Supported Versions

Security fixes are made on the latest `main` branch.

## Reporting a Vulnerability

Please do not open a public issue for a potential vulnerability. Use GitHub's
private vulnerability reporting for this repository when available, or contact
the repository owner privately through GitHub.

Include the affected revision, the hardware or network access required, a clear
reproduction path, and the potential impact. Do not include passwords, Matter
pairing codes, Wi-Fi credentials, private keys, or customer device data.
